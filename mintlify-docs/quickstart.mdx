FinalRun's three AI agent skills let you go from "I have an app repo" to a green mobile test run inside your AI coding agent. The skills work in **Claude Code, Cursor, and Codex** (and any other agent that supports the skills protocol). You describe what you want covered in plain English; the skills generate the YAML, run the CLI, read the artifacts on failure, and apply fixes until the run is green.

This guide assumes you've finished [Installation](/installation) and have an [API key set](/configuration/cloud-api-key).

<Steps>
  <Step title="Open your app's repo in your AI coding agent">
    Launch Claude Code, Cursor, or Codex from the root of the mobile app repository you want to test. The skills run from chat — you don't have to switch tools.

    <Note>
      The FinalRun installer registers all three skills globally. If you skipped that step, install them with:

      ```bash
      npx skills add final-run/finalrun-agent -g
      ```

      See [AI Agent Skills](/running/ai-agent-skills) for the full skill reference.
    </Note>
  </Step>

  <Step title="Generate tests with /finalrun-generate-test">
    Ask the agent to generate tests for a feature in your app. The skill reads your source code, infers the package name or bundle ID, scaffolds `.finalrun/config.yaml` and `.finalrun/env/<env>.yaml`, and writes YAML specs grouped by feature.

    ```bash
    /finalrun-generate-test Generate tests for the authentication feature — cover login with valid credentials, login with wrong password, and logout
    ```

    The skill proposes a test plan first and waits for your approval before writing files. Once approved, it writes specs under `.finalrun/tests/<feature>/`, a matching suite under `.finalrun/suites/`, and runs `finalrun check` to validate the workspace.
  </Step>

  <Step title="Run and fix with /finalrun-test-and-fix">
    Hand the running and triage to the orchestrator skill. It executes the new tests, reads `result.json`, screenshots, and device logs on failure, decides whether the bug is in the **app code** or the **test**, applies the narrowest fix, and re-runs until the run is green.

    ```bash
    /finalrun-test-and-fix Verify and fix the auth feature end-to-end on Android
    ```

    Prefer to run without orchestration? Use the lower-level skill instead — it just runs the tests and reports the outcome:

    ```bash
    /finalrun-use-cli Run the auth tests on Android
    ```
  </Step>

  <Step title="View the report">
    List recent runs and open the local report UI to see screenshots, video, and device logs:

    ```bash
    finalrun runs
    finalrun start-server
    ```
  </Step>
</Steps>

## Make it automatic

Add a definition-of-done rule to your `AGENTS.md` that triggers `/finalrun-test-and-fix` after every UI change. Your agent then treats a passing FinalRun run as a requirement for closing any UI task — see [Auto-triggering FinalRun](/running/ai-agent-skills#auto-triggering-finalrun-after-feature-work).
